# 優化 Review Checklist

每次優化完成後，依序走一遍這 10 大類，逐項確認。

---

## 1. 執行緒安全與競態條件

> 多執行緒環境下的共享資源存取是否安全？

- [ ] 「檢查＋操作」是否為原子操作？（避免 TOCTOU：檢查通過後被其他執行緒搶先）
- [ ] 同一資源是否被多個執行緒重複操作？（如雙重平倉、重複寫入）
- [ ] 共享物件是否有鎖保護？（如 LearningEngine、RiskManager）
- [ ] 鎖是否有嵌套導致死鎖風險？（確認 lock 取得順序一致）

---

## 2. 失敗回滾機制

> 操作中途失敗時，系統狀態是否能正確回復？

- [ ] 預登記的資源（倉位、餘額）在後續步驟失敗時是否有回滾？
- [ ] 驗證失敗（倉位太小、數量為零）後是否清理已登記的狀態？
- [ ] 外部呼叫（下單 API）失敗後是否回滾內部狀態？

---

## 3. 信號 / 數據一致性

> 消費端讀到的數據是否新鮮、完整、不重複？

- [ ] 共享數據是否有過期檢測機制？（生產端崩潰後消費端不該用陳舊數據）
- [ ] 同一信號是否會被重複消費？（需過濾已處理 / 已持倉的信號）
- [ ] 已持倉幣種是否在**所有入口**都被過濾？（短線和趨勢都要檢查）
- [ ] 部分寫入問題：消費端是否可能讀到半更新的數據？（確認鎖粒度）

---

## 4. 掃描頻率與數據特性匹配

> 掃描間隔是否符合數據的實際更新頻率和信號時效性？

- [ ] K 線閉合週期 vs 掃描間隔是否對齊？（1M → 45-60s，4H → 10-20min）
- [ ] 掃描耗時是否已計入間隔設計？（掃描 15s + sleep 45s = 實際 60s）
- [ ] 不同策略的頻率需求差異是否被架構支持？（不要用交替掃描耦合不同時間尺度）
- [ ] 是否有重要時間節點需要特殊觸發？（如 4H K 線閉合整點）

---

## 5. 資源重複與浪費

> 多個模組是否在做重複的工作？

- [ ] 多個執行緒是否獨立呼叫同一 API 取相同數據？（應共用結果）
- [ ] 共享數據（btc_trend、tickers）是否有快取 / 共用機制？
- [ ] API rate limit 是否有估算？改動後是否仍在安全範圍？
- [ ] 連接池大小是否匹配並行數？（pool_maxsize vs MAX_WORKERS）

---

## 6. 通知與日誌完整性

> 所有決策路徑（執行 / 跳過）和所有策略類型是否都有記錄和通知？

- [ ] 每個 skip 原因是否都有日誌？（分數不足、倉位太小、cooldown、持倉上限...）
- [ ] 每個 open / close 事件是否都有日誌？（含 PnL、持倉時間）
- [ ] 所有策略類型是否都涵蓋通知？（逐一確認：短線 Discord、趨勢 Discord、開倉通知、平倉通知）
- [ ] 通知格式是否能區分策略類型？（加標籤避免混淆）
- [ ] 新增的功能路徑是否有接入既有的通知管道？（新掃描、新策略、新事件都要檢查）
- [ ] 前端儀表板是否同步顯示新增的數據？（新欄位、新狀態、新標籤）

---

## 7. 位置 / 倉位數據完整性

> Position dict 是否包含所有下游邏輯需要的欄位？

- [ ] 新增策略類型時，position dict 是否同步新增必要欄位？（如 strategy_type、atr）
- [ ] 所有判斷分支是否覆蓋所有策略名稱？（用集合比對而非單一字串比對）
- [ ] standalone bot 和 web_app bot 的 position 結構是否一致？

---

## 8. 狀態初始化與持久化

> 系統重啟後是否能正確恢復？新增狀態是否有初始值？

- [ ] 新增的 state key 是否有預設值？（避免 KeyError）
- [ ] 持倉和餘額在重啟後是否能從日誌恢復？
- [ ] 首次啟動時的邊界條件是否正確處理？（如 timestamp=0 的行為）

---

## 9. 觸發邏輯的邊界處理

> 定時觸發或條件觸發是否有防重複和邊界保護？

- [ ] 時間窗口內是否會重複觸發？（需要去重標記）
- [ ] 觸發後是否有標記防止同週期重複？
- [ ] 計數器邏輯在長時間運行後是否會溢出或漂移？

---

## 10. 程式碼衛生與一致性

> 多處修改後，全檔案是否保持一致？

- [ ] 舊常數是否有殘留？（grep 確認所有 sleep / 間隔用的是新常數）
- [ ] import 是否有未使用或位置不當的？（移到頂層，移除未用的）
- [ ] 多個呼叫端（web_app / standalone bot）是否同步更新？
- [ ] Unicode / 編碼是否正確？（特別是中文字串）
- [ ] 前端顯示值是否與後端實際值一致？
